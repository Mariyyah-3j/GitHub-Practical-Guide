# 🚀 رحلة تعلم Git & GitHub

## 📌 فكرة المشروع

هذا المشروع عبارة عن تجربة تعليمية تأخذ المستخدم في **رحلة متدرجة** لفهم Git و GitHub من الأساسيات حتى العمل الجماعي والمساهمة في مشاريع مفتوحة المصدر.

صممنا التجربة كمسار تعليمي يبدأ من Command Line وينتهي بالاحتراف في GitHub.
وفي نهاية الرحلة، يحصل المستخدم على 🎁 **كنز تعليمي**:
ملصق يحتوي على أهم أوامر Git كمرجع سريع للطباعة أو الاستخدام الرقمي.

# 🟢 المرحلة الأولى: التحكم في النظام (Command Line)

تبدأ الرحلة بفهم:
- ما هو Command Line ولماذا يستخدمه المحترفون
- الفرق بين File و Folder
- كيفية تنظيم الملفات والمجلدات
- التنقل داخل الجهاز باستخدام الأوامر
- إنشاء المجلدات والملفات
- نسخ ونقل وحذف الملفات
- الفرق بين Absolute Path و Relative Path

🎯 الهدف من هذه المرحلة:
التحكم الكامل في بيئة العمل قبل الانتقال لإدارة الإصدارات.

---

# 🔵 المرحلة الثانية: فهم Version Control

في هذه المرحلة يتعلم المستخدم:
- ما هو Version Control ولماذا هو أساسي
- الفرق بين Git و GitHub
- لماذا يعتمد المطورون على Git في إدارة مشاريعهم
- كيف يمنع Git ضياع العمل أو فقدان التعديلات

🎯 الهدف من هذه المرحلة:
فهم أهمية تتبع التعديلات قبل التطبيق العملي.

---

# 🟣 المرحلة الثالثة: أول مشروع باستخدام Git

في هذه المرحلة يبدأ التطبيق:
- إنشاء مشروع جديد باستخدام:
  - `git init`
- إضافة الملفات:
  - `git add`
- حفظ التعديلات:
  - `git commit`
- مراجعة السجل:
  - `git log`
- فهم Workflow الصحيح عند إضافة تعديلات جديدة
- حل مشكلة AutoSave

🎯 الهدف من هذه المرحلة:
إدارة نسخ المشروع باحترافية محليًا.

---

# 🟡 المرحلة الرابعة: الانطلاق إلى GitHub

في هذه المرحلة يتم:

- إنشاء حساب على GitHub
- إنشاء Repository خاص
- ربط المشروع المحلي بـ GitHub
- رفع المشروع باستخدام `git push`
- فهم Profile README
- إنشاء واجهة احترافية للحساب
- التعرف على Stars و Gist
- فهم Projects و Organizations و Codespaces

🎯 الهدف من هذه المرحلة:
بناء حضور احترافي على GitHub.

---

# 🔴 المرحلة الخامسة: العمل الجماعي والاحتراف

في هذه المرحلة يتعلم المستخدم:

- Branching وأهميته
- إنشاء Pull Request
- المراجعة قبل الدمج
- استخدام `git clone`
- الفرق بين Collaborator و Fork
- تنفيذ Fork Workflow
- إرسال Pull Request لمشاريع خارجية
- فهم آلية المساهمة في Open Source

🎯 الهدف من هذه المرحلة:
تطبيق رحلة المطور الحقيقية في بيئة عمل جماعية.

---

# 🏆 الكنز: ملصق الأوامر

في نهاية الرحلة، يحصل المستخدم على ملصق تعليمي يحتوي على:

- Git Basics
- Branching Commands
- Remote Commands
- Fix & Undo Commands
- Workflow Summary

الملصق متوفر بصيغ:
- PNG
- PDF للطباعة

# ⚙️ آلية العمل داخل المشروع

تم الالتزام بالقواعد التالية:

- المستودع خاص (Private)
- عدم التعديل المباشر على `main`

- إنشاء Pull Request قبل الدمج
